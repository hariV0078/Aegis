PrivacyForge Workflow

End-to-End App Flow

This page shows the full PrivacyForge workflow: user input, agent creation, token_map-preserving execution, local SQLite persistence, and the Midnight proof seam for on-chain anchoring.

Data Flow

Key Guarantees
PII first: input is anonymized before model or tool execution.
State integrity: token_map stays with workflow state through every node.
Local storage: agents and runs are persisted in SQLite only.
Midnight seam: proof anchoring is optional and testnet-oriented, with a local fallback.
